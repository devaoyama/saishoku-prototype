## 1. ユーザー画面

| No | 画面名 | 概要 |
| --- | --- | --- |
| 1 | トップ（即時面談ハブ） | オンライン中パートナーを強調表示。「今すぐ話す」「予約する」へ遷移 |
| 2 | パートナー詳細 | 写真・紹介文・タグ・対応形式表示。即時面談／予約へ |
| 3 | 情報入力（即時面談・予約 共通） | 面談開始/予約確定の前に、**メール/電話番号等を入力**。ここで **「自動でアカウントを作成する」チェック**を表示 |
| 4 | 即時面談開始（Zoom/電話案内） | Zoom参加URL表示・参加ボタン または `tel:`リンクによる電話発信・注意事項（顔出し不要OK等） |
| 5 | 予約画面（空き枠表示） | 空き枠表示→日時選択→確定（確定前にNo.3へ誘導してもOK） |
| 6 | 予約完了（Zoom案内） | 日時・Zoom参加URL・当日の案内を表示（メールでも送付） |
| 7 | ログイン | 既存ユーザー向けログイン（メール起点想定） |
| 8 | マイページ（最小） | 次回面談、予約履歴、基本情報の確認/編集（MVP最小） |
| 9 | 利用規約・免責（軽量） | 個人情報・録音録画・禁止事項など |

## 2. 管理画面（admin）

管理画面は**パートナー向け・運営向けを統一**した1つの管理画面とする。

- **AIによる履歴書作成支援**が可能
- **AIによる人間的市場価値診断**が可能
- **CRM**（基本情報・ログ・マッチングデータ等）で候補者・求人・選考を一元管理

### 2.1 パートナー・予約管理

| No | 画面名 | 概要 |
| --- | --- | --- |
| 10 | パートナーダッシュボード | 即時対応ON/OFF、対応形式（音声のみ/ビデオOK）、本日の予約一覧 |
| 11 | 予約枠管理 | 空き枠の登録・編集（Googleカレンダー連携は任意） |
| 12 | おすすめ求人表示 | タグマッチングに基づく候補者への「おすすめ求人」表示（マッチ率表示） |

### 2.2 全体管理

| No | 画面名 | 概要 |
| --- | --- | --- |
| 13 | 全体予約一覧 | 全パートナーの予約を一覧で確認・運用対応 |
| 14 | 運営ダッシュボード | 全体状況の概要表示、各種管理画面への導線 |

### 2.3 候補者管理（CRM）

| No | 画面名 | 概要 |
| --- | --- | --- |
| 15 | 候補者一覧 | 候補者の一覧表示、検索・フィルタリング |
| 16 | 候補者詳細 | 基本情報、対応履歴、選考状況の表示・編集。**この候補者で**おすすめ求人・AI履歴書作成・市場価値診断へ遷移可能 |
| 17 | 候補者タグ編集 | 性格・志向性タグの選択・編集（`#ハイテンション` `#営業志向` 等） |
| 18 | 面談ログ入力 | 面談後の記録入力（タグ選択式、フリーテキスト最小化） |

### 2.4 求人管理

| No | 画面名 | 概要 |
| --- | --- | --- |
| 19 | 求人一覧 | 求人の一覧表示、検索・フィルタリング、新規作成への導線 |
| 19-2 | 求人作成 | 企業名・職種・条件・タグ・必須要件・福利厚生を入力して求人を新規登録 |
| 20 | 求人詳細・編集 | 企業情報、求人内容、カルチャータグの編集 |
| 21 | 求人タグ編集 | 企業カルチャータグの選択・編集（`#体育会系` `#静かな職場` 等） |

### 2.5 マッチング・選考管理

| No | 画面名 | 概要 |

### 2.6 タグマスタ管理

## 画面遷移概要

```
[ユーザーフロー]
トップ画面(1) → パートナー詳細(2) → 情報入力(3) → 即時面談開始(4) / 予約画面(5) → 予約完了(6)
                                                    ↓
                                            ログイン(7) → マイページ(8)

[管理フロー（admin）]
運営ダッシュボード(14) → パートナー・予約(10-12) / 候補者CRM(15-18) / 求人(19-21) / マッチング・選考(22-24)
                      → タグマスタ(25-26) / AI機能(27-29) / 分析・マーケ(30-32) / 外部連携(33-34)
```
